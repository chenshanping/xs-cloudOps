# AI 提示词模板 - 项目需求文档与数据库设计

```bash
我想开发一个生态养殖管理系统，请帮我完成以下工作：

## 一、系统命名
请帮我设计：
- 系统中文名（2-5个字，简洁有意义）
- 系统英文名（驼峰命名，适合做项目名）
- login_slogan（登录页标语，10-15字）
- login_desc（系统描述，30-50字）
- login_features（4个核心特性，每个2-4字）

## 二、功能需求
系统包含以下角色和功能：

### 角色1：管理员
牲畜分类：管理所有牲畜分类信息
牲畜批次：管理所有牲畜批次信息，需要选择养殖区域（同一种类型的同一个批次一次记录）
养殖区域：管理所有养殖区域信息
饲料信息：管理所有饲料信息
喂养记录：管理所有饲料使用记录信息
疫苗接种：管理所有疫苗接种信息，需要选择动物和接种日期
疾病上报记录：管理所有疾病记录
养殖知识：管理所有养殖知识文章信息，由管理员上传，养殖户可以学习
认证信息：管理所有养殖户的认证信息，审核养殖户认证信息，只有认证通过的养殖户才能正常使用养殖户功能
公告信息：管理所有公告信息


### 角色2：养殖户
养殖知识：浏览所有养殖知识信息
牲畜信息：管理当前登录养殖户的养殖牲畜信息
养殖区域：管理当前登录养殖户的养殖区域信息
饲料信息：管理当前登录养殖户的饲料信息
喂养记录：管理当前登录养殖户的饲料使用记录信息
疫苗接种：管理当前登录养殖户的疫苗接种信息，需要选择牲畜批次和接种日期
疾病记录：管理当前登录养殖户的疾病记录

## 三、输出要求
1. 需求文档（Markdown格式）
   - 项目概述
   - 用户角色说明
   - 功能模块详细说明
   - 数据字典
   - API接口规划

2. 数据库设计（MySQL SQL脚本）
   - 完整的建表语句
   - 字段注释
   - 索引设计
   - 初始化数据
3.项目有内置代码生成器 结合来完成 最后一部分需要写代码生成器如何使用生成业务模块
4.所有文件生成到一个md文件里面
```
