# 改进：导入导出数据验证和模板说明

## 改进内容

### 1. 下载模板时添加详细说明 ✨

**改进前：**
- 只有简单的表头（字段名 + 必填/选填标记）
- 没有任何使用说明
- 用户不知道每个字段可以填什么值

**改进后：**
- ✅ 表头添加批注（鼠标悬停显示详细说明）
- ✅ 数据字典字段：显示字典类型和可选值
- ✅ 关联字段：说明需要填写的字段名称
- ✅ 开关字段：显示可选的两个值
- ✅ 数字/时间字段：说明格式要求
- ✅ 添加Excel下拉数据验证（防止输入错误）

**示例：**

| 字段 | 批注说明 | 下拉验证 |
|------|---------|---------|
| 状态(必填) | 数据字典: common_status<br>请从下拉列表中选择 | ✅ 启用, 禁用 |
| 产品类型(必填) | 关联字段: name<br>请填写已存在的产品类型名称 | ❌ |
| 是否推荐(选填) | 可选值: 是, 否<br>请从下拉列表中选择 | ✅ 是, 否 |
| 价格(必填) | 请填写数字 | ❌ |
| 创建时间(选填) | 时间格式: 2006-01-02 15:04:05 或 2006-01-02 | ❌ |

### 2. 导入时严格验证数据 ✨

**改进前：**
- 数据字典值不存在时，静默失败或使用原值
- 关联对象不存在时，只显示简单错误
- 没有显示有效值列表
- 多个错误分散在不同行

**改进后：**
- ✅ 数据字典值验证：不存在时拒绝导入，并显示所有有效值
- ✅ 固定选项验证：不在选项列表中时拒绝导入
- ✅ 关联对象验证：不存在时拒绝导入，提示先创建
- ✅ 开关字段验证：只接受指定的两个值
- ✅ 必填字段验证：空值时拒绝导入
- ✅ 错误信息聚合：一行的所有错误一起显示
- ✅ 跳过空行：自动忽略完全空白的行

**错误提示示例：**

```
导入完成
成功: 8 条，失败: 2 条

错误详情:
第3行: 状态值"已停用"不存在，有效值: 启用, 禁用; 产品类型"测试类型"不存在，请先创建该产品类型
第5行: 价格为必填项; 是否推荐值"可能"无效，有效值: 是, 否
```

## 技术实现

### 1. Excel工具类增强

**新增方法：**

```go
// SetHeadersWithComments 设置表头并添加批注
func (e *ExcelExporter) SetHeadersWithComments(headers []string, comments []string) error

// AddDataValidation 为指定列添加下拉数据验证
func (e *ExcelExporter) AddDataValidation(colIndex int, options []string, startRow, endRow int) error
```

**使用示例：**

```go
// 设置表头和批注
headers := []string{"状态(必填)", "产品类型(必填)"}
comments := []string{
    "数据字典: common_status\\n请从下拉列表中选择",
    "关联字段: name\\n请填写已存在的产品类型名称",
}
exporter.SetHeadersWithComments(headers, comments)

// 添加下拉验证
options := []string{"启用", "禁用"}
exporter.AddDataValidation(0, options, 2, 1000) // 第0列，从第2行到第1000行
```

### 2. 导入验证逻辑

**验证流程：**

1. **跳过空行**：检查整行是否为空
2. **逐字段验证**：
   - 数据字典：查询有效值列表，不存在时记录错误
   - 固定选项：检查是否在选项列表中
   - 关联对象：查询数据库，不存在时记录错误
   - 开关字段：只接受指定的两个值
   - 必填字段：检查是否为空
3. **错误聚合**：收集该行所有错误
4. **有错误则跳过**：不保存有错误的行
5. **返回详细结果**：成功数、失败数、错误列表

**代码示例：**

```go
// 数据字典验证
if cellValue != "" {
    dictValue := service.Dict.GetDictValue("common_status", cellValue)
    if dictValue == "" {
        // 获取所有有效值
        dictList, _ := service.Dict.GetDictDataList("common_status")
        validValues := make([]string, 0, len(dictList))
        for _, item := range dictList {
            validValues = append(validValues, item.Label)
        }
        rowErrors = append(rowErrors, fmt.Sprintf("状态值\"%s\"不存在，有效值: %s", 
            cellValue, strings.Join(validValues, ", ")))
        hasError = true
    }
}
```

## 配置说明

### 字典服务方法

需要在 `service/dict.go` 中添加：

```go
// GetDictDataList 根据字典类型获取字典数据列表（用于模板生成）
func (s *DictService) GetDictDataList(dictType string) ([]model.SysDictData, error) {
    return s.GetDictDataByType(dictType)
}
```

### 模板配置

在生成器配置中，字段配置会自动识别：
- `DictType`: 数据字典类型
- `SelectOptions`: 固定选项列表
- `SwitchValues`: 开关字段的两个值
- `IsRequired`: 是否必填
- `Relations`: 关联关系

## 注意事项

### 1. 数据字典必须存在

如果使用了数据字典字段，必须先在系统中创建对应的字典类型和数据：

```sql
-- 检查字典是否存在
SELECT * FROM sys_dict_type WHERE type = 'common_status';
SELECT * FROM sys_dict_data WHERE dict_type = 'common_status';
```

### 2. 关联对象必须先创建

导入时如果引用了关联对象，该对象必须已存在：

```sql
-- 检查关联对象是否存在
SELECT * FROM product_type WHERE name = '测试类型';
```

### 3. Excel下拉验证的限制

- 下拉列表最多支持255个字符
- 如果选项太多，下拉验证可能失败
- 此时只显示批注说明，不添加下拉验证

### 4. 批注在不同Excel版本中的显示

- Excel 2016+: 完美支持
- WPS: 支持，但样式可能略有不同
- LibreOffice: 支持，但需要手动显示批注

## 常见问题

### Q1: 下拉列表不显示？

**原因：** 选项太多或Excel版本不支持

**解决：** 查看批注说明，手动输入有效值

### Q2: 批注不显示？

**原因：** Excel设置问题

**解决：** 
1. Excel选项 -> 高级 -> 显示 -> 批注和指示器
2. 或右键单元格 -> 显示/隐藏批注

### Q3: 导入时提示"有效值"列表太长？

**原因：** 数据字典选项太多

**解决：** 
1. 精简数据字典选项
2. 或使用代码导入而不是Excel导入

### Q4: 关联对象名称有重复怎么办？

**原因：** 关联表中有多条同名记录

**解决：** 
1. 确保关联字段（如name）唯一
2. 或使用ID而不是名称进行关联
