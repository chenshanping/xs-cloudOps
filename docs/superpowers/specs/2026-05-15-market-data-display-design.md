# 股票与基金数据展示模块设计

## 1. 背景

当前项目是一个基于 Go + Gin + GORM + Vue 3 + Ant Design Vue 的后台管理框架，适合承载中后台型业务模块。本次需求是在现有框架内新增一个“投研数据”模块，用于展示 A 股和公募基金的数据，不做前台站点，不做自动交易，不做 AI 推荐结论。

本设计文档只覆盖 V1 的后台展示能力，为后续实现计划和代码落地提供边界。

## 2. 已确认决策

- 仅做后台展示版，不做前台用户站点
- 市场范围为 A 股 + 公募基金
- 数据刷新方式为日线收盘后同步
- 数据源优先使用 AKShare 零预算方案
- V1 不做 AI 解读、不做荐股荐基、不做自动交易、不做复杂回测
- V1 采用本地落库展示，不走页面实时直连第三方接口

## 3. 目标

在当前后台框架中交付一个最小可用闭环，完成以下能力：

- 展示 A 股股票列表
- 展示股票详情和近段时间日线走势
- 展示公募基金列表
- 展示基金详情和近段时间净值走势
- 提供后台手动同步入口
- 支持收盘后定时同步
- 保留同步日志，便于排查数据问题

## 4. 非目标

以下内容明确不在 V1 范围内：

- 前台用户端页面
- 分钟线、Tick、盘口、实时推送
- 自动买卖、券商下单、资金管理
- AI 生成投资建议
- 回测、选股打分、组合优化
- 港股、美股、债券等其他市场与品类扩展
- 人工维护股票/基金主数据的新增、编辑、删除

## 5. 方案比较

### 方案 A：实时转发型

页面打开后由后端实时调用第三方数据源并直接返回前端，不做本地持久化。

优点：

- 实现速度快
- 初期表结构最少

缺点：

- 页面稳定性依赖第三方接口
- 容易受限流、超时、字段波动影响
- 不适合列表筛选、历史图表和管理后台查询

结论：

不推荐作为正式业务模块方案。

### 方案 B：本地落库型

定时或手动从 AKShare 同步股票和基金数据到本地数据库，后台页面只查询本地库。

优点：

- 页面稳定
- 查询快
- 易于做筛选、分页、图表和日志追踪
- 最符合当前后台框架的业务形态

缺点：

- 数据不是实时的
- 需要同步任务和落库逻辑

结论：

推荐作为 V1 方案。

### 方案 C：混合型

列表和历史图走本地库，部分详情补充信息走第三方临时查询。

优点：

- 信息更丰富

缺点：

- 复杂度提升明显
- 本地数据与临时查询数据可能不一致
- V1 容易超范围

结论：

可作为后续增强方向，不纳入 V1。

## 6. 推荐方案

V1 采用“本地落库型”。

总体思路：

- 使用 AKShare 作为数据源
- 使用独立采集脚本或轻量采集层拉取外部数据
- 由 Go 后端负责同步调度、数据入库、权限控制、页面查询和日志记录
- 后台页面仅查询本地 MySQL 数据

该方案与当前仓库现有的管理后台、权限系统、定时任务能力和 CRUD 扩展模式最匹配。

## 9. 同步流程设计

### 9.1 同步类型

V1 提供四类同步动作：

- 股票基础信息同步
- 股票日线同步
- 基金基础信息同步
- 基金净值同步

### 9.2 触发方式

提供两种入口：

- 定时任务：收盘后自动同步
- 手动触发：后台点击按钮立即执行

### 9.3 同步策略

- 基础信息：按主键做新增或更新
- 日线与净值：按代码 + 日期做增量同步
- 每次同步都写日志

### 9.4 错误处理

- 单条失败不阻断整批任务
- 聚合记录失败数量和摘要
- 页面可查看最近一次结果

### 9.5 数据采集方式

由于 AKShare 主要位于 Python 生态，V1 推荐：

- 使用独立采集脚本负责请求外部数据
- Go 后端负责调度脚本、解析结果、写入数据库

这样可以避免把外部数据抓取逻辑直接耦合到 Go 主服务内部。

## 13. 风险与约束

### 13.1 数据源稳定性

AKShare 基于公开数据源封装，字段或可用性可能波动。系统需要接受偶发同步失败，并通过同步日志保证可排查。

### 13.2 数据延迟

本方案不是实时行情方案。V1 默认接受“收盘后同步”带来的延迟。

### 13.3 存储边界

V1 只做日线，不做分钟线和 Tick，避免 MySQL 表规模快速膨胀。

### 13.4 字段标准化

采集层需要先完成字段映射，再入库，避免把外部接口字段格式直接扩散到前后端业务层。

### 13.5 系统边界

本模块定位是“数据展示与辅助判断”，不是投资建议系统，不应输出确定性买卖结论。

## 15. 最终结论

在当前 go-base 框架上开发“股票与基金数据展示模块”是可行的。

最合适的 V1 方案是：

- 后台展示版
- A 股 + 公募基金
- 收盘后日线同步
- AKShare 零预算数据源
- 本地落库展示
- 列表、详情、图表、同步任务四类页面闭环

这个方案实现成本可控、边界清晰，也最符合当前仓库的架构和管理后台模式。
