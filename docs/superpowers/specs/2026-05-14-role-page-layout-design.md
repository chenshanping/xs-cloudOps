# 角色管理页面布局收敛设计

## 目标

在不改后端接口、不重写权限逻辑的前提下，按 `docs/superpowers/role-permission-split-mockup.html` 完善角色管理页面布局，统一角色列表与三个权限抽屉的视觉层级和交互入口。

## 范围

- 调整角色管理列表页头部、摘要卡片、搜索区和操作列布局。
- 保留 `编辑 / 删除 / 更多` 的操作结构。
- 将菜单权限、API 权限、数据权限继续放在 `更多` 下拉中。
- 统一菜单权限、API 权限、数据权限、关联用户抽屉的标题区和响应式宽度。

## 非目标

- 不修改任何后端接口、权限接口协议和保存逻辑。
- 不重构权限树、API 分组或数据权限表单内部算法。
- 不新增新的统计接口或复杂筛选能力。

## 设计决策

### 1. 列表页保持现有 ProTable

继续复用现有 `ProTable`，只调整外层页头和表格内容展现，避免为视觉一致性重写整页表格壳。

### 2. 搜索采用前端本地筛选

角色列表当前直接返回全量数据，因此搜索只筛选角色名称和角色编码，不引入后端查询参数变更。

### 3. 摘要卡片只展示可靠数据

摘要区仅展示当前接口数据中可直接可靠计算的指标：

- 角色总数
- 启用角色数
- 超管角色数
- 关联用户总数

### 4. 抽屉统一外壳，不改内部业务逻辑

三个权限抽屉和关联用户抽屉统一：

- 标题区角色上下文
- 副标题说明
- `maskClosable=false`
- 响应式宽度和窄屏全宽

## 验证

- 回读修改后的页面和抽屉文件，确认结构与导入正确。
- 使用搜索/符号查找确认新增状态和方法已接入页面模板。
- 不运行 `npm run build` / `npm run typecheck`，按项目规则交给用户做 dev server 点击验证。
