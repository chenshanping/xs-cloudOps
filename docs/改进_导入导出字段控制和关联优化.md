# 改进：导入导出字段控制和关联优化

## 改进内容

### 1. 产品类型options接口过滤status=1 ✅

**问题：** `/productType/options` 接口返回所有产品类型，包括禁用的

**解决：** 已在service层添加 `Where("status=1")` 过滤

**代码位置：** `go-base-server/service/productType.go:182`

```go
err := global.DB.Model(&model.ProductType{}).
    Select("id, `" + displayField + "` as name").
    Where("status=1").  // ✅ 只返回启用的
    Order("id ASC").
    Find(&results).Error
```

### 2. 下载模板时调用options接口 ✅

**问题：** 下载模板时直接查询关联表，没有过滤status等条件

**解决：** 修改模板，优先使用options接口（如果配置了 `UseOptionsApi`）

**代码位置：** `go-base-server/generator/template/api.tpl`

```go
{{- if .UseOptionsApi}}
// 使用options接口获取选项（已过滤status等条件）
optionsList, _ := service.{{.RelatedModel}}.Get{{.RelatedModel}}Options("{{.DisplayField}}", "", "", false, 0)
options := make([]string, 0, len(optionsList))
for _, item := range optionsList {
    if name, ok := item["name"].(string); ok {
        options = append(options, name)
    }
}
{{- else}}
// 直接查询关联表
var relatedList []model.{{.RelatedModel}}
if err := global.DB.Find(&relatedList).Error; err == nil {
    // ...
}
{{- end}}
```

**优点：**
- ✅ 自动过滤status=1
- ✅ 不需要每次都判断status
- ✅ 统一使用options接口的逻辑

### 3. 添加字段配置：是否可导入导出 ✅

**新增配置：** `IsExportable` 字段

**代码位置：** `go-base-server/generator/config.go`

```go
type ColumnConfig struct {
    // ...
    IsListVisible bool   `json:"is_list_visible"` // 是否列表显示
    IsFormVisible bool   `json:"is_form_visible"` // 是否表单显示
    IsExportable  bool   `json:"is_exportable"`   // 是否可导入导出 ✨ 新增
    IsSortable    bool   `json:"is_sortable"`     // 是否可排序
    // ...
}
```

**使用方式：**

在代码生成器配置中，为每个字段设置 `is_exportable`：

```json
{
  "columns": [
    {
      "column_name": "name",
      "is_exportable": true  // 可导入导出
    },
    {
      "column_name": "password",
      "is_exportable": false  // 不可导入导出（敏感字段）
    }
  ]
}
```

**生成的代码：**

模板会生成 `ExportColumns` 列表，只包含 `IsExportable=true` 的字段：

```go
// generator.go
exportColumns := make([]ColumnConfig, 0)
for _, col := range g.Config.Columns {
    if col.IsExportable && !belongsToForeignKeys[col.ColumnName] {
        exportColumns = append(exportColumns, col)
    }
}
```

**默认行为：**

如果没有配置 `IsExportable`（旧配置），默认使用 `FormColumns`（表单字段）。

### 4. 关联关系导入导出优化 🚧

**需求：**
- 点击左侧产品类型时，导出的产品数据应该自动包含该类型
- 下载模板时不显示产品类型字段（因为已经选定了）
- 导入时自动使用当前选定的产品类型

**实现方案：**

#### 方案A：前端传递参数（推荐）

**导出时：**
```typescript
// 前端
const handleExport = async () => {
  const params = {
    ...searchForm,  // 包含 product_type_id
  }
  await exportProduct(params)
}
```

**下载模板时：**
```typescript
// 前端
const handleDownloadTemplate = async () => {
  const params = {
    product_type_id: selectedCategoryId.value  // 传递当前选定的类型
  }
  await downloadTemplateProduct(params)
}
```

**后端处理：**
```go
// api.tpl
func (a *{{.ModelName}}Api) DownloadTemplate{{.ModelName}}(c *gin.Context) {
    // 获取筛选参数
    productTypeId := c.Query("product_type_id")
    
    // 如果指定了产品类型，不显示产品类型字段
    skipFields := make(map[string]bool)
    if productTypeId != "" {
        skipFields["product_type_id"] = true
    }
    
    // 设置表头时跳过指定字段
    headers := []string{}
    for _, col := range exportColumns {
        if !skipFields[col.ColumnName] {
            headers = append(headers, col.Comment)
        }
    }
    // ...
}
```

**导入时：**
```go
// api.tpl
func (a *{{.ModelName}}Api) Import{{.ModelName}}(c *gin.Context) {
    // 获取筛选参数
    productTypeId := c.PostForm("product_type_id")
    
    // 导入时自动设置产品类型
    for i, row := range rows {
        var createReq request.Create{{.ModelName}}Request
        // ... 解析其他字段
        
        // 如果指定了产品类型，自动设置
        if productTypeId != "" {
            if id, err := strconv.ParseUint(productTypeId, 10, 64); err == nil {
                createReq.ProductTypeId = uint(id)
            }
        }
        
        // 保存数据
        service.{{.ModelName}}.Create{{.ModelName}}(&createReq, userID)
    }
}
```

#### 方案B：配置字段属性（更灵活）

添加字段配置：`ExcludeFromTemplateWhenFiltered`

```go
type ColumnConfig struct {
    // ...
    ExcludeFromTemplateWhenFiltered bool `json:"exclude_from_template_when_filtered"` // 筛选时从模板中排除
}
```

**使用场景：**
- 产品的 `product_type_id` 字段设置为 `true`
- 当前端传递了 `product_type_id` 参数时，模板中不显示该字段
- 导入时自动使用传递的值

## 注意事项

### 1. 默认行为

如果没有配置 `IsExportable`，建议默认行为：
- 表单字段（`IsFormVisible=true`）默认可导出
- 敏感字段（password、token等）默认不可导出

### 2. 关联字段

belongsTo关联的外键字段（如 `product_type_id`）：
- 不直接导出外键ID
- 导出关联对象的显示字段（如 `product_type.name`）
- 导入时通过显示字段查询ID

### 3. 文件字段

文件字段（file、files、images）：
- 导出时导出文件URL
- 导入时不支持（需要单独上传）
- 建议设置 `IsExportable=false`

### 4. 富文本字段

富文本字段（editor）：
- 导出时导出HTML内容
- 导入时支持HTML
- 但Excel显示效果不好，建议设置 `IsExportable=false`
