## Context

当前仓库已经具备：

- 用户管理列表、左侧部门树筛选、批量启用/禁用/删除
- 单条重置密码接口
- 系统配置 `sys_config` 读写链路
- 基于部门数据范围的用户可管理性校验

本次改动需要同时触达 `server/`、`web/`、权限元数据、初始化补齐和 SQL 修补脚本，但范围仍然保持在后台用户管理局部，不扩展到更大的密码体系。

约束：

- 继续沿用 Gin + GORM + Casbin + Redis + Vue3 + Ant Design Vue 现有模式
- 不新增依赖
- 非平凡交互继续保持当前页面的局部工具栏/确认弹窗模式

## Decisions

### 1. 默认密码来源统一收敛到系统配置

决策：

- 新增系统配置键 `user_default_password`
- 单条重置密码和批量重置密码都由后端读取该配置决定目标密码
- 若配置缺失，回退到兼容默认值 `123456`

原因：

- 避免前端写死密码
- 让单条与批量行为保持一致
- 与当前仓库已有 `sys_config` 机制一致

备选方案：

- 前端每次传入密码：会把默认密码逻辑散落到页面层，排除
- 新增单独密码策略表：超出本轮范围

### 2. 批量重置密码接口只传用户 ID 列表

决策：

- 新增 `PUT /users/batch-password`
- 请求体仅包含 `ids`

原因：

- 默认密码已由服务端配置决定，前端无需传密码
- 前后端责任清晰，接口更稳定

### 3. 重置密码权限语义保持与当前单条操作一致

决策：

- 单条重置继续复用 `EnsureUserManageable`
- 批量重置继续复用 `EnsureUsersManageable`
- 不额外新增“不能重置自己”或“受保护管理员不可重置”的新规则

原因：

- 本轮目标是把默认密码来源和批量闭环补齐，不额外改变业务口径
- 现有单条重置密码已经形成可管理范围语义，保持一致最稳

### 4. 左侧部门树用受控展开 key，而不是一次性默认属性

决策：

- 页面维护 `expandedKeys`
- 初始化按全部部门节点计算完整 key 集合，首次全展开
- 通过“全部展开 / 全部收缩”按钮直接切换该状态

原因：

- 既满足默认全展开，也能支持后续可控收起
- 比只加 `defaultExpandAll` 更适合和按钮交互配合

### 5. 用户默认密码配置使用密码框并提供随机生成

决策：

- 基础配置页新增密码框输入
- 同行新增“随机生成”按钮
- 生成规则固定为 12 位字母数字

原因：

- 与 SMTP 授权码、AI Key 等现有敏感配置输入体验一致
- 随机生成能减少手填弱密码
- 不引入特殊字符，避免复制/录入摩擦

### 6. 误选风险在当前页面局部闭环内消除

决策：

- 搜索、重置、切换部门树、翻页、批量操作成功后清空 `selectedRowKeys`
- 单条/批量重置密码都增加确认弹窗

原因：

- 这是当前页面已经暴露出来的直接闭环缺口
- 风险低，收益直接

## Backend Design

- `request` 新增批量重置密码请求模型，仅含 `Ids []uint`
- `api/v1/user.go`
  - 单条重置密码接口改为不再绑定密码字段
  - 新增批量重置密码接口
- `service/user/user.go`
  - 抽出读取默认密码配置的 helper
  - 单条重置密码改为读取配置后调用统一重置逻辑
  - 批量重置密码在通过可管理性校验后，统一对目标用户写入同一个新哈希密码
- `router/modules/user.go`
  - 注册新接口
- `initialize/db_tables.go`
  - 补齐 `user_default_password`
  - 补齐批量重置按钮权限、API 元数据和角色继承授权
- `server/sql/`
  - 新增一个仅针对本功能的增量脚本，duplicate-safe 地补配置键、按钮权限、API、角色授权和 Casbin 策略

## Frontend Design

- `web/src/store/config.ts`
  - 增加 `user_default_password` key 和默认值
- `SystemConfig.vue`
  - 新增密码框配置项和随机生成按钮
  - 随机生成只回填当前表单，保存后才真正生效
- `user/index.vue`
  - 左侧树改为受控展开
  - 树头增加展开/收缩操作
  - 未绑定部门在树与表格里统一红色标签
  - 新增批量重置密码按钮
  - 单条/批量重置增加确认弹窗
  - 在搜索、切换、翻页、批量成功后清空勾选
- `api/user.ts`
  - 新增批量重置密码 API wrapper
  - 单条重置密码改为无密码参数调用

## Risks

- 旧环境没有 `user_default_password` 配置：通过默认值和初始化补齐兜底
- 批量重置密码属于敏感操作：通过确认弹窗和后端统一读配置降低误操作
- SQL 脚本可能运行在部分已升级环境：用 `NOT EXISTS` 与已有权限继承模式保证幂等

## Verification

- 后端集成测试验证单条/批量重置密码都读取系统配置，并校验越权拒绝
- 前端通过构建与页面验证确认树展开、标签强调、确认弹窗和勾选清空行为
